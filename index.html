📖 Step-by-Step Guide: S3 Static Website Hosting

1

Create an S3 Bucket

  • Go to AWS Console → S3
  • Click "Create bucket"
  • Enter a unique bucket name (e.g., juan-portfolio-2026)
  • Select a region (e.g., ap-southeast-1 for Singapore)
  • Uncheck the "Block all public access" checkbox
  • Acknowledge the warning and click "Create bucket"
2

Enable Static Website Hosting

  • Open the bucket → Properties tab
  • Scroll down to "Static website hosting"
  • Click "Edit" → Enable
  • Index document: index.html
  • Error document: error.html
  • Click "Save changes"
3

Set the Bucket Policy

  • Go to the Permissions tab
  • Under "Bucket policy", click "Edit"
  • Paste the following policy (replace YOUR-BUCKET-NAME):
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
        }
    ]
}
4

Upload Your Website Files

  • Go to the Objects tab → Click "Upload"
  • Upload the following files:
    • index.html — main page
    • error.html — custom error page
    • style.css — stylesheet
    • images/ — folder for images
5

Get the Website URL and Submit

  • Go to the Properties tab
  • Scroll down to "Static website hosting"
  • Copy the Bucket website endpoint
  • Paste it in the Checker tab and submit! 🎉
💡 Tip: The URL follows this format:
http://bucket-name.s3-website-region.amazonaws.com

🔧 Common Issues & Fixes

403 Forbidden

"Block all public access" is still checked, or the bucket policy is missing.

404 Not Found

No index.html in the bucket, or the filename is wrong (it's case-sensitive!).

Website doesn't load

Static Website Hosting is not enabled under Properties.

Wrong URL format

Use the website endpoint, not the S3 object URL. Find it under Properties → Static website hosting.
