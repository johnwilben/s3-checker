📖 Step-by-Step Guide: S3 Static Website Hosting

1

Gumawa ng S3 Bucket

  • Pumunta sa AWS Console → S3
  • Click "Create bucket"
  • Lagyan ng unique na bucket name (e.g., juan-portfolio-2026)
  • Piliin ang region (e.g., ap-southeast-1 para Singapore)
  • Uncheck ang "Block all public access" checkbox
  • Acknowledge ang warning at click "Create bucket"
2

I-enable ang Static Website Hosting

  • Buksan ang bucket → Properties tab
  • Scroll down sa "Static website hosting"
  • Click "Edit" → Enable
  • Index document: index.html
  • Error document: error.html
  • Click "Save changes"
3

I-set ang Bucket Policy

  • Pumunta sa Permissions tab
  • Sa "Bucket policy", click "Edit"
  • I-paste ang policy na ito (palitan ang YOUR-BUCKET-NAME):
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
        }
    ]
}
4

I-upload ang Website Files

  • Pumunta sa Objects tab → Click "Upload"
  • I-upload ang mga files:
    • index.html — main page
    • error.html — custom error page
    • style.css — stylesheet
    • images/ — folder para sa images
5

Kunin ang Website URL at I-submit

  • Pumunta sa Properties tab
  • Scroll down sa "Static website hosting"
  • Kopyahin ang Bucket website endpoint
  • I-paste sa Checker tab at i-submit! 🎉
💡 Tip: Ang URL ay ganito ang format:
http://bucket-name.s3-website-region.amazonaws.com

🔧 Common Issues & Fixes

403 Forbidden

Hindi naka-uncheck ang "Block all public access" o walang bucket policy.

404 Not Found

Walang index.html sa bucket o mali ang filename (case-sensitive!).

Website hindi nag-lo-load

Hindi naka-enable ang Static Website Hosting sa Properties.

Mali ang URL format

Gamitin ang website endpoint, hindi ang S3 object URL.
